# Folder Structure

## Product

Nexus

Universal AI Integration Platform

---

# Purpose

This document defines the official monorepo structure for Nexus.

All code must follow this structure.

This document acts as the source of truth for:

* Service boundaries
* Package ownership
* Connector organization
* Runtime separation

---

# Dependency Rules

apps

↓

packages

↓

services

↓

connectors

---

Allowed:

```text
apps → packages

apps → services

services → packages

connectors → connector-sdk
```

---

Forbidden:

```text
connectors → database

connectors → oauth storage

connectors → other connectors

mcp-runtime → connector internals
```

# Design Principle

Every folder should have a single responsibility.

Connectors execute provider logic.

Execution Engine executes tools.

OAuth manages credentials.

Registry manages tools.

MCP exposes tools.

These boundaries must never be violated.
